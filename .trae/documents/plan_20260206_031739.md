### 挑战与解决方案
火山引擎 (Volcengine) 的 API 通常需要 **AWS v4 风格的签名 (Signature V4)**。Python SDK 帮用户处理了这些，但在 Node.js 中我们需要自己实现或者引入一个轻量级的签名库。

为了简化实现并确保成功率，我将使用 `aws4` 库（它完全兼容火山引擎的签名算法），手动构造 HTTP 请求。

### 实施计划

#### 1. 安装签名依赖
*   安装 `aws4`：用于生成 API 签名头 (Authorization, X-Date 等)。

#### 2. 实现 Jimeng Provider (`lib/ai-providers/jimeng.ts`)
*   **配置**: 读取环境变量 `JIMENG_AK` (Access Key) 和 `JIMENG_SK` (Secret Key)。
*   **API 映射**:
    *   **Host**: `visual.volcengineapi.com`
    *   **Region**: `cn-north-1`
    *   **Action**: `CVProcess` (这是火山视觉服务的通用 Action)
    *   **Version**: `2022-08-31`
*   **请求构造**:
    *   提交任务 (`submit_task`): 构造 POST 请求，带上 `req_key: jimeng_t2i_v40`。
    *   查询结果 (`get_result`): 构造 GET/POST 请求，轮询 `status: done`。
