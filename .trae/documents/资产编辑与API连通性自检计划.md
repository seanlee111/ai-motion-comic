## 现状诊断
- 资产库目前仅支持创建/删除，后端接口只有 GET/POST/DELETE（缺少更新接口）。
- API 连通性需要更直观的一键检测入口，目前只有手动尝试生成才知道是否通。

## 方案概述
### 1) 资产库支持编辑
- 新增「资产详情编辑」能力：点击侧边栏资产卡片打开编辑面板/弹窗。
- 支持修改：名称、描述、类型、替换图片（并保留原图或覆盖）。
- 后端补充 PATCH/PUT：更新资产元数据与图片资源。
- 前端更新：资产卡片增加“编辑”入口，CreateAssetDialog 复用为 Edit 模式。

### 2) API 连通性自检
- 新增「API 自检」按钮（Header/Settings 里），一键检测：Jimeng、Kling、LLM。
- 新增服务端 /api/health（或 /api/check）路由：
  - 返回各服务可用性与错误原因（缺 key/鉴权失败/接口不可达）。
  - 可选：支持轻量“空 Prompt”干跑/鉴权探测（不消耗配额）。
- 前端展示清晰结果：每个服务一行状态（✅/❌ + 失败原因）。

## 具体实现步骤
1. 扩展后端资产 API：新增更新接口（PATCH/PUT），支持更新 name/description/type/图片。
2. 前端资产库新增编辑入口：点击资产卡片进入编辑界面，支持修改与保存。
3. 新增 API 自检路由：聚合检测 Jimeng/Kling/LLM 是否可用。
4. 前端新增自检 UI：一键测试并显示结果。
