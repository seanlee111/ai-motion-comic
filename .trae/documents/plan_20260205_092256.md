### 1. 多模型 API 预留 (API Expansion)
我们将扩展全局设置和状态管理，预留以下 API 接口位置：
*   **Jimeng (即梦/Dreamina)**: 字节跳动出品。
*   **Kling (可灵)**: 快手出品，视频生成强项。
*   **Minimax (海螺)**: 视频与文本模型。
*   **Nanobanana**: 预留位置。
*   **LLM API (新增)**: 用于 AI 剧本分析（推荐使用 Minimax 或 OpenAI 兼容接口）。

### 2. 新增功能：AI 剧本创作 (AI Script Writer)
我们将增加一个新的页面 `/script`，用于智能生成分镜脚本。
*   **输入**: 简单的故事梗概（例如：“两个武侠在竹林里决斗，风吹叶落，最后一人倒下”）。
*   **处理**: 调用 AI 模型（LLM）自动分析并拆解分镜。
*   **输出**: 自动生成分镜列表，包含以下字段：
    *   **时长 (Duration)**: 预估该镜头的秒数。
    *   **角色 (Characters)**: 自动提取或分配角色。
    *   **运镜与动作 (Movement/Action)**: 描述镜头运动（如推拉摇移）和角色动作。
    *   **一键导入**: 将生成的剧本一键转换为“分镜表”数据。

### 3. 分镜工作台升级 (Storyboard Upgrade)
针对您的需求“生成第一帧和最后一帧”，我们将改造分镜卡片：
*   **双图模式**: 每个分镜卡片现在显示两个图片槽位 —— **Start Frame (首帧)** 和 **End Frame (尾帧)**。
*   **生成逻辑**:
    *   点击生成时，AI 会根据“动作描述”分别生成起始状态和结束状态的图片。
    *   这将为后续生成视频（Image-to-Video）提供完美的 Keyframes 输入。
